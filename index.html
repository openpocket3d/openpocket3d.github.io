OpenPocket3D is a plug-and-play framework for extending open-vocabulary 3D instance segmentation models to the open-ended setting.

Abstract

Open-ended 3D scene understanding requires recognizing object semantics without a predefined query vocabulary, while open-vocabulary 3D instance segmentation still often assigns semantics by comparing language-aligned 3D representations with text queries supplied at inference time. Recent open-ended 3D methods reduce this dependence by generating open-ended classes from visual inputs, but they can aggregate weakly visible views or query a language model for each proposal, introducing noisy evidence and extra cost. We propose OpenPocket3D, a modular framework that converts frozen open-vocabulary 3D instance segmentation backbones into open-ended predictors without retraining the backbone or training a separate semantic decoder. For each 3D proposal, OpenPocket3D ranks views by visibility, localizes generated classes using internal attention maps from a 2D vision-language model, and constructs an instance-wise pocket vocabulary by retaining classes supported by projected 3D masks. It outputs a prediction by combining 3D-text semantic similarity from the frozen backbone with geometric consistency from mask projection. By reusing precomputed 3D masks and instance features, OpenPocket3D avoids additional point-level dense feature lifting and proposal-wise language-model queries during open-ended prediction, while remaining compatible with multiple backbones and VLM sources. Experiments on ScanNet200, ScanNet++, and Replica show that OpenPocket3D achieves 21.4 AP on ScanNet200 and state-of-the-art performance among open-ended methods on most reported ScanNet200 and ScanNet++ metrics.
